(2) Faça um manual de Instalação e configuração de sistema operacionais. Windows ( 11 e 10) e Linux.

Este guia apresenta os passos principais para a instalação e configuração básica dos sistemas operacionais Windows 10, Windows 11 e Linux (considerando distribuições populares como Ubuntu e Mint).

Observações Importantes:

  • Backup: Antes de qualquer instalação, faça um backup completo dos seus dados importantes!
  • Requisitos de Sistema: Verifique os requisitos mínimos e recomendados de cada sistema operacional para garantir o bom funcionamento no seu hardware.
  • Mídia de Instalação: Você precisará de um pendrive bootável ou DVD com os arquivos de instalação do sistema operacional desejado.

Windows (11 e 10)

Os processos de instalação do Windows 10 e 11 são bastante similares. As principais diferenças residem na interface e em alguns requisitos de hardware (especialmente para o Windows 11).

    Preparação:

    • Criar Mídia de Instalação:
      Windows 10:
    Utilize a "Media Creation Tool" da Microsoft (disponível no site oficial) para criar um pendrive bootável ou gravar um DVD.
      Windows 11:
    Utilize a "Media Creation Tool" da Microsoft (disponível no site oficial) para criar um pendrive bootável. Geralmente, a instalação via DVD não é recomendada.
    • Configurar o Boot:
      Acesse a BIOS/UEFI
    do seu computador (geralmente pressionando Del, F2, F12 ou outra tecla durante a inicialização - verifique o manual da sua placa-mãe).
      Altere a ordem de boot
    para que o pendrive ou DVD seja a primeira opção.
    • Requisitos do Windows 11 (Ponto de Atenção):
    TPM 2.0 (Trusted Platform Module) ativado na BIOS/UEFI. Secure Boot ativado na BIOS/UEFI. Processador compatível (verifique a lista de processadores suportados pela Microsoft).
    • Instalação:
    • Iniciar a Instalação:
    Salve as alterações na BIOS/UEFI e reinicie o computador com a mídia de instalação conectada. A instalação do Windows deverá iniciar automaticamente.
    • Idioma, Formato de Hora e Teclado:
    Selecione suas preferências de idioma, formato de hora e teclado. Clique em "Avançar".
    • Instalar Agora:
    Clique em "Instalar agora".
    • Ativação (Opcional):
    Se você tiver uma chave de produto, insira-a. Caso contrário, pode selecionar "Não tenho a chave do produto" para ativar posteriormente (o Windows funcionará com funcionalidades limitadas até a ativação).
    • Termos de Licença:
    Leia e aceite os termos de licença. Clique em "Avançar".
    • Tipo de Instalação:
    Selecione "Personalizada: Instalar apenas o Windows (avançado)". Onde você quer instalar o Windows? Selecione a partição onde você deseja instalar o Windows. Cuidado: Se você deseja formatar uma partição existente, selecione-a e clique em "Formatar". Se você tem um disco novo, pode selecionar o espaço não alocado e clicar em "Novo" para criar partições. Clique em "Avançar".
    • Copiando arquivos do Windows...:
    Aguarde a conclusão do processo de cópia e instalação dos arquivos. O computador será reiniciado várias vezes.
    • Configurações Iniciais:
    Após a reinicialização, você será guiado por algumas telas de configuração: Região e idioma. Layout do teclado. Conexão de rede (Wi-Fi ou Ethernet). Nome do dispositivo (opcional). Como você gostaria de configurar? (Uso pessoal ou organizacional). Conta Microsoft ou conta local. Definir um PIN (opcional). Configurações de privacidade. Cortana (assistente virtual - opcional).
    • Finalização:
    Aguarde a preparação da área de trabalho.
    • Configuração Pós-Instalação:
    • Instalar Drivers:
    O Windows geralmente instala drivers básicos, mas é recomendável instalar os drivers mais recentes do site do fabricante do seu hardware (placa-mãe, placa de vídeo, etc.).
    • Ativar o Windows (se necessário):
    Vá em "Configurações" > "Atualização e Segurança" > "Ativação".
    • Windows Update:
    Execute o Windows Update para instalar as atualizações mais recentes de segurança e funcionalidades.
      Instalar Programas:
    Instale os softwares que você utiliza.
    • Configurar Perfil de Usuário:
    Personalize as configurações da sua conta de usuário (papel de parede, temas, etc.).
    • Configurar Firewall e Segurança:
    Verifique as configurações do Firewall do Windows e instale um software antivírus, se desejar.

II. Linux (Exemplo: Ubuntu/Mint)

O processo de instalação do Linux pode variar um pouco entre distribuições, mas os passos gerais são semelhantes. Ubuntu e Mint são conhecidos por terem instaladores gráficos amigáveis.

    A Preparação:
    Baixar a ISO:
Acesse o site oficial da distribuição Linux que você escolheu (ex: Ubuntu, Linux Mint) e baixe a imagem ISO.
    Criar Mídia de Instalação:
Utilize um software como Rufus (Windows), Etcher (multiplataforma) ou o utilitário de criação de disco de inicialização do próprio Linux (se você já tiver um sistema Linux) para criar um pendrive bootável com a ISO baixada.
    Configurar o Boot:
Acesse a BIOS/UEFI do seu computador e altere a ordem de boot para que o pendrive seja a primeira opção.
    Instalação:
    Iniciar a Instalação:
Salve as alterações na BIOS/UEFI e reinicie o computador com o pendrive conectado. Um menu de boot do Linux deverá aparecer. Selecione "Install [Nome da Distribuição]" ou similar. Algumas distribuições oferecem a opção de "Experimentar" antes de instalar.
    Idioma:
Selecione o idioma desejado para a instalação e para o sistema operacional.
    Conexão Wi-Fi (Opcional):
Selecione sua rede Wi-Fi e insira a senha, se necessário.
    Atualizações e Outros Softwares:
Geralmente, você terá opções para baixar atualizações durante a instalação e instalar softwares de terceiros (como codecs multimídia). Escolha de acordo com suas preferências.
    Tipo de Instalação:
Apagar disco e instalar [Nome da Distribuição]: Esta opção formatará todo o disco e instalará o Linux. Use com cautela, pois todos os dados serão perdidos. Instalação personalizada (avançado): Permite criar ou redimensionar partições manualmente. Recomendado para usuários mais experientes ou para dual boot. Instalar ao lado de [Outro Sistema Operacional]: Se outro sistema operacional for detectado, essa opção permitirá criar uma nova partição para o Linux.
    Particionamento (se "Instalação personalizada" for escolhida):
Você precisará criar pelo menos duas partições: :/ (root): Onde o sistema operacional será instalado. Recomenda-se um tamanho mínimo de 20-30 GB. :swap: Área de troca (memória virtual). O tamanho recomendado geralmente é igual à quantidade de RAM ou o dobro, dependendo da quantidade de RAM e do uso. Opcionalmente, você pode criar uma partição /home separada para seus arquivos pessoais.
    Fuso Horário:
Selecione seu fuso horário.
    Informações do Usuário:
Digite seu nome, nome de usuário, senha e nome do computador.
    Aguardar a Instalação:
O instalador copiará os arquivos e configurará o sistema.
    Reiniciar:
Após a conclusão, remova a mídia de instalação e reinicie o computador.
    C. Configuração Pós-Instalação:
    Atualizar o Sistema:
Abra o terminal e execute os seguintes comandos (para distribuições baseadas em Debian/Ubuntu): Bash sudo apt update sudo apt upgrade
    Instalar Drivers (se necessário):
Alguns drivers proprietários (como os de placa de vídeo Nvidia) podem precisar ser instalados manualmente através do "Gerenciador de Drivers" ou da linha de comando.
    Instalar Programas:
Utilize a loja de aplicativos (ex: Ubuntu Software Center, Mint Install) ou o gerenciador de pacotes (apt no terminal) para instalar os softwares desejados.
    Configurar o Ambiente de Trabalho:
Personalize a aparência, temas, ícones e outras configurações do ambiente de trabalho.
    Configurar Rede e Conexões:
Verifique as configurações de rede e configure outras conexões, como impressoras.
    Configurar Segurança:
Explore as configurações de firewall (ex: UFW) e outras ferramentas de segurança.

(3) 10 Questões com 5 alternativas cada AD: Grupo de Trabalho e Delegação de Controle de UO.

1. Qual o principal objetivo de se criar um Grupo de Trabalho (Global, Domain Local ou Universal) no Active Directory?

  1. Implementar políticas de segurança específicas para usuários dentro de uma Unidade Organizacional (UO).
  2. Simplificar a atribuição de permissões a múltiplos usuários e/ou computadores simultaneamente.
  3. Delegar o controle administrativo de uma Unidade Organizacional para um usuário ou grupo específico.
  4. Restringir o acesso de determinados usuários a recursos compartilhados na rede
    5. .
  5. Organizar objetos (usuários, computadores, grupos) dentro de um domínio de forma lógica.

2. Qual escopo de grupo no Active Directory possui a menor abrangência e só pode conter membros do seu próprio domínio?

  1. Global.
  2. Universal.
  3. Domain Local.
  4. Grupo de segurança.
  5. Grupo de distribuição.

3. Em qual cenário o escopo de grupo Universal é mais indicado para ser utilizado?

  1. Quando se precisa conceder acesso a um recurso que reside no mesmo domínio dos usuários.
  2. Quando se deseja agrupar usuários de diferentes domínios em uma floresta para conceder acesso a um recurso em um domínio específico.
  3. Quando a necessidade é gerenciar permissões apenas dentro de uma Unidade Organizacional.
  4. Quando se precisa enviar e-mails para um grande número de destinatários internos e externos.
  5. Quando se quer aplicar uma GPO (Objeto de Política de Grupo) a um conjunto específico de computadores.

4. O que significa delegar o controle de uma Unidade Organizacional (UO) no Active Directory?

  1. Mover objetos (usuários, computadores) para dentro da UO.
  2. Aplicar políticas de senha específicas para os usuários dentro da UO.
  3. Conceder a um usuário ou grupo específico permissões administrativas para gerenciar os objetos contidos naquela UO.
  4. Auditar as ações realizadas pelos usuários dentro da UO.
  5. Impedir que administradores de outros domínios visualizem os objetos da UO.

5. Qual ferramenta do Windows Server é utilizada para realizar a delegação de controle de uma Unidade Organizacional?

  1. Gerenciador do Servidor.
  2. Usuários e Computadores do Active Directory (dsa.msc).
  3. Política de Grupo Management Console (gpmc.msc).
  4. Powershell com módulos do Active Directory.
  5. Monitor de Desempenho.

6. Ao delegar o controle de uma UO, quais tipos de tarefas administrativas podem ser concedidas a um usuário ou grupo?

  1. Apenas a capacidade de visualizar os objetos dentro da UO.
  2. Apenas a capacidade de redefinir senhas de usuários na UO.
  3. Uma variedade de tarefas, como criar, excluir, modificar usuários, grupos, computadores e redefinir senhas, dependendo das permissões delegadas.
  4. Apenas a capacidade de mover objetos para outras UOs.
  5. Apenas a capacidade de aplicar Objetos de Política de Grupo (GPOs) à UO.

7. Qual o princípio de segurança fundamental que a delegação de controle de UO busca promover?

  1. Segurança por obscuridade.
  2. Princípio do menor privilégio.
  3. Defesa em profundidade.
  4. Segregação de funções.
  5. Auditoria constante.

8. Quais são os benefícios de se delegar o controle administrativo de UOs em um ambiente Active Directory?

  1. Aumenta a carga de trabalho do administrador principal do domínio.
  2. Centraliza todas as tarefas administrativas em uma única conta.
  3. Melhora a eficiência e agilidade na gestão de usuários e recursos dentro de departamentos ou equipes específicas.
  4. Diminui a segurança do ambiente, pois mais usuários possuem permissões elevadas.
  5. Dificulta a implementação de políticas de segurança consistentes.

9. Qual a relação entre Grupos de Trabalho e a delegação de controle de UO?

  1. Grupos de trabalho são necessários para que a delegação de controle de UO funcione corretamente.
  2. A delegação de controle de UO permite gerenciar os membros de um grupo de trabalho.
  3. Grupos de trabalho podem ser utilizados para conceder as permissões delegadas em uma UO a um conjunto de usuários.
  4. Não existe relação direta entre grupos de trabalho e delegação de controle de UO. Grupos de trabalho são um tipo específico de delegação de controle de UO.

10. Ao delegar o controle de uma UO, é possível conceder permissões para gerenciar as associações de membros de um grupo contido nessa UO?

  1. Não, a delegação de controle de UO só permite gerenciar os objetos diretamente contidos na UO.
  2. Sim, é possível delegar permissões específicas para adicionar e remover membros de grupos dentro da UO.
  3. Apenas o administrador do domínio possui permissão para gerenciar membros de grupos.
  4. Isso depende do escopo do grupo (Global, Domain Local ou Universal).
  5. A delegação de controle de UO se aplica apenas a usuários e computadores, não a grupos.
