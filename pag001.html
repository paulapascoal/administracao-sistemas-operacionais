(2) Por que a virtualização é útil

A virtualização é útil porque permite criar versões virtuais de recursos físicos, como servidores, computadores, sistemas operacionais e aplicativos. Por exemplo:

  • Economia de dinheiro: Em vez de comprar vários computadores físicos, você pode ter vários virtuais rodando em um único hardware mais potente. Isso reduz custos com compra, energia e espaço físico.
  • Melhor uso dos recursos: A virtualização permite aproveitar ao máximo a capacidade de um servidor. Se um servidor físico está ocioso, a virtualização permite rodar outras máquinas virtuais nele, evitando desperdício.
  • Flexibilidade e agilidade: Criar e remover máquinas virtuais é muito mais rápido e fácil do que configurar um servidor físico do zero. Isso facilita testar novos softwares, criar ambientes de desenvolvimento e escalar recursos rapidamente.
  • Facilidade de gerenciamento: É mais fácil gerenciar vários ambientes virtuais centralizadamente do que lidar com vários hardwares separados
  • Maior disponibilidade: A virtualização facilita a implementação de soluções de backup e recuperação, tornando os sistemas mais resilientes a falhas. Se um servidor físico falhar, as máquinas virtuais podem ser rapidamente movidas para outro.

A virtualização torna a infraestrutura de TI mais eficiente, flexível e econômica. É como ter vários computadores dentro de um só, com todas as vantagens que isso traz.

(3) Terminologia: O que é Host OS, guest OS e VM?

  • Host OS (Sistema Operacional Hospedeiro): É o sistema operacional instalado diretamente no hardware físico do computador. É o sistema operacional "base" que está rodando a máquina. Pense nele como o sistema operacional principal do seu computador.
  • Guest OS (Sistema Operacional Convidado): É o sistema operacional instalado dentro de uma máquina virtual. Ele roda como um software dentro do Host OS. Você pode ter vários Guest OS diferentes rodando simultaneamente no mesmo Host OS. Pense neles como sistemas operacionais "dentro" do seu sistema operacional principal.
  • VM (Virtual Machine ou Máquina Virtual): É uma simulação de um computador criada por software. Ela possui seu próprio hardware virtual (CPU, memória, disco, etc.) e pode executar um sistema operacional (o Guest OS) e aplicativos como se fosse um computador físico separado. Pense nela como um computador completo que existe apenas como um arquivo no seu computador real.

(4) Segurança da Informação: Senha

O objetivo principal de uma senha é garantir que apenas pessoas ou sistemas autorizados tenham acesso a informações, recursos ou funcionalidades específicas.

Uma senha forte é crucial para a segurança, pois dificulta que invasores consigam adivinhá-la ou quebrá-la por meio de diferentes métodos (como ataques de força bruta ou dicionário).

(5) Crie uma senha segura.

Precisa ter letras, números e caracteres especiais de preferencialmente acima de 10 digitos.
