(1)Informações relevantes sobre Windows Server

Windows Server é uma linha de sistemas operacionais da Microsoft projetada especificamente para servidores. Ele se diferencia das versões desktop do Windows por oferecer recursos avançados para gerenciar redes, hospedar serviços, armazenar dados e executar aplicações empresariais.

1. Qual das seguintes funcionalidades é uma característica principal do Windows Server?

  1. Interface gráfica otimizada para uso doméstico.
  2. Ferramentas avançadas para gerenciamento de redes e servidores.
  3. Foco exclusivo em aplicações de produtividade pessoal.
  4. Ausência de recursos de segurança integrados.
  5. Limitação na capacidade de hospedar serviços de rede.

(2)O que é AD e ADDS?

  • AD (Active Directory): É um serviço de diretório da Microsoft utilizado em redes de domínio para gerenciar informações sobre usuários, computadores, grupos, políticas de segurança e outros objetos. Pense nele como um banco de dados centralizado que organiza e controla o acesso aos recursos da rede. Ele fornece autenticação (verificação de identidade) e autorização (permissão de acesso) para usuários e computadores.
  • ADDS (Active Directory Domain Services): É a função de servidor no Windows Server que implementa e hospeda o serviço de diretório Active Directory. Em outras palavras, o ADDS é o componente do Windows Server que você instala para criar e gerenciar um domínio do Active Directory. Ele fornece os serviços essenciais para autenticação, autorização e gerenciamento centralizado.

2. Qual é a principal função do Active Directory (AD) em uma rede de domínio?

  1. Fornecer acesso à internet para os usuários.
  2. Gerenciar a impressão de documentos na rede.
  3. Centralizar o gerenciamento de usuários, computadores e políticas de segurança.
  4. Monitorar o tráfego de rede em tempo real.
  5. Realizar backups automáticos dos dados dos usuários.

(3)Os tipos de relação de confiança do AD?

    • Confiança Transitiva: É o tipo padrão de confiança estabelecido entre domínios em uma mesma floresta do Active Directory. Se o Domínio A confia no Domínio B, e o Domínio B confia no Domínio C, então o Domínio A confia automaticamente no Domínio C 1 (a confiança "transita").
      • Confiança Não Transitiva (ou Externa): É uma relação de confiança unidirecional ou bidirecional estabelecida entre domínios que não estão na mesma floresta. Essa confiança precisa ser configurada explicitamente e não é transitiva.
    • Confiança de Atalho (Shortcut Trust): Criada dentro de uma floresta com uma estrutura de árvore complexa para otimizar o processo de autenticação entre domínios que estão logicamente distantes na hierarquia. Ela cria um caminho de confiança mais direto.
    • Confiança de Floresta (Forest Trust): Permite a confiança entre duas florestas inteiras do Active Directory. Pode ser unidirecional ou bidirecional e permite que usuários de uma floresta autentiquem-se e acessem recursos na outra.

3. Qual tipo de relação de confiança no Active Directory é estabelecida automaticamente entre domínios dentro da mesma floresta, permitindo que a confiança se estenda através da hierarquia?

  1. Confiança Externa Confiança de Atalho
  2. Confiança de Floresta
  3. Confiança Transitiva
  4. Confiança Unidirecional
